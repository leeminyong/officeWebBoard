# AGENTS.md

이 파일은 Codex 같은 코딩 도구가 이 프로젝트에서 작업할 때 따라야 하는 지침입니다.

## 대화 방식

- 기본적으로 한국어로 답변합니다.
- 사용자가 웹개발을 한 번도 해보지 않았다고 생각하고 쉽게 설명합니다.
- 어려운 개발 용어는 가능한 한 피합니다.
- 꼭 필요한 개발 용어를 써야 할 때는, 그 단어가 무슨 뜻인지 짧게 풀어서 설명합니다.

## 코드 수정 방식

- 코드를 수정할 때마다, 수정한 코드 근처에 주석을 추가합니다.
- 주석은 웹개발을 처음 보는 사람도 이해할 수 있게 자세히 적습니다.
- 주석에는 “이 코드가 어떤 기능을 하는지”와 “왜 이렇게 바꿨는지”를 설명합니다.
- 주석을 달 때는 해당 파일에서 쓰는 주석 문법도 간단히 설명합니다.
- 예를 들어 HTML은 `<!-- 설명 -->`, CSS는 `/* 설명 */`, JavaScript는 `// 설명` 또는 `/* 설명 */` 형태를 사용한다고 알려줍니다.
- 화면에 보이는 부분을 바꾼 경우, 사용자가 화면에서 무엇을 다르게 보게 되는지도 설명합니다.
- 서버 동작을 바꾼 경우, 어떤 요청, 파일, 데이터 흐름이 영향을 받는지 설명합니다.
- 주석은 실제로 수정한 기능과 관련된 내용만 적고, 의미 없는 설명은 넣지 않습니다.

## 프로젝트 구조

- 웹서버는 `D:\officeWebBoard` 폴더에서 실행됩니다.
- 서버의 중심 파일은 `server.js`입니다.
- 브라우저에서 보이는 화면 파일들은 `public` 폴더 안에 있습니다.
- 웹사이트 접속 주소는 보통 `http://192.168.0.139:3000`입니다.
- 수동 실행과 종료를 위한 보조 파일은 `D:\100.WebBoard` 폴더에 있습니다.

## 작업 안전 규칙

- 사용자가 직접 바꾼 내용을 함부로 삭제하거나 되돌리지 않습니다.
- 코드를 바꾸기 전에는 주변 코드를 먼저 읽고, 기존 방식에 맞춰 수정합니다.
- 큰 구조를 한 번에 바꾸기보다, 필요한 부분만 작고 안전하게 수정합니다.
