# 변경 기록과 주석 작성 규칙

## 앞으로의 수정 규칙

- 코드를 수정할 때는 수정한 코드 바로 위에 짧은 주석을 남깁니다.
- 주석에는 "왜 수정했는지"와 "어떻게 수정했는지"를 간단히 적습니다.
- 웹개발 문법을 몰라도 흐름을 따라갈 수 있도록 어려운 용어는 되도록 피합니다.

## 2026-04-29

### HTTPS에서 HTTP로 되돌림

- 사무실 노트북마다 인증서를 설치해야 하는 불편이 있어 게시판 접속 방식을 다시 HTTP로 변경했습니다.
- 현재 접속 주소는 `http://192.168.56.102:3000/?board=maintenance`입니다.
- `server.js`에서 HTTPS 인증서 사용 코드를 제거하고 Express 기본 `app.listen` 방식으로 되돌렸습니다.

### 댓글, 첨부파일, 수정 기능 주석 보강

- `server.js`에 댓글 등록/삭제, 첨부파일 저장/다운로드/삭제, 게시글 수정 API 설명 주석을 추가했습니다.
- `public/view.html`에 댓글 작성, 댓글 파일 첨부, 파일 다운로드/삭제, 수정 화면 이동 설명 주석을 추가했습니다.
- `public/write.html`에 수정 모드 판별, 기존 첨부파일 삭제 표시, 새 첨부파일 추가, 등록/수정 저장 설명 주석을 추가했습니다.

### 내부망 접속 주소 변경

- `localhost` 대신 같은 사무실 내부망에서 접속할 수 있도록 서버 주소를 `0.0.0.0`으로 열었습니다.
- 현재 접속 주소는 `192.168.56.102` 기준으로 안내되도록 했습니다.
- Windows 방화벽에 `TCP 3000` 인바운드 허용 규칙을 추가했습니다.

### HTTPS 적용

- `https://192.168.56.102:3000/?board=maintenance` 주소로 접속할 수 있도록 서버를 HTTPS 방식으로 바꿨습니다.
- `D:\websource\certs\onepass-local.pfx` 자체 서명 인증서를 사용합니다.
- 자체 서명 인증서라서 다른 노트북 브라우저에서는 처음 접속할 때 보안 경고가 뜰 수 있습니다.
- Chrome 보안 경고를 줄이기 위해 노트북에 설치할 수 있는 `D:\websource\certs\onepass-local.cer` 파일을 만들었습니다.
- 설치 방법은 `D:\websource\HTTPS_CERT_INSTALL.md`에 정리했습니다.
