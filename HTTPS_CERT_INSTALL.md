# HTTPS 인증서 설치 방법

## 왜 설치하나요?

`https://192.168.56.102:3000` 주소는 내부망에서 쓰는 자체 서명 인증서를 사용합니다.
Chrome은 이 인증서를 처음에는 믿지 않기 때문에 `NET::ERR_CERT_AUTHORITY_INVALID` 경고를 보여줍니다.

## 설치할 파일

아래 파일을 접속할 노트북으로 옮겨서 설치합니다.

`D:\websource\certs\onepass-local.cer`

## Windows 노트북 설치 순서

### 쉬운 방법

1. `D:\websource\certs` 폴더에서 아래 두 파일을 노트북으로 옮깁니다.

`onepass-local.cer`

`install-onepass-cert.cmd`

2. 노트북에서 `install-onepass-cert.cmd`를 마우스 오른쪽 버튼으로 누릅니다.
3. `관리자 권한으로 실행`을 선택합니다.
4. Chrome을 완전히 닫았다가 다시 엽니다.
5. 아래 주소로 접속합니다.

`https://192.168.56.102:3000/?board=maintenance`

### 직접 설치 방법

1. `onepass-local.cer` 파일을 더블클릭합니다.
2. `인증서 설치`를 누릅니다.
3. `현재 사용자` 또는 `로컬 컴퓨터`를 선택합니다.
4. `모든 인증서를 다음 저장소에 저장`을 선택합니다.
5. `찾아보기`를 누르고 `신뢰할 수 있는 루트 인증 기관`을 선택합니다.
6. `다음`을 누른 뒤 `마침`을 누릅니다.
7. Chrome을 완전히 닫았다가 다시 엽니다.
8. 아래 주소로 접속합니다.

`https://192.168.56.102:3000/?board=maintenance`

## 주의

- 이 인증서는 사무실 내부망 테스트용입니다.
- 이 PC의 IP가 바뀌면 인증서를 새로 만들어야 할 수 있습니다.
- 외부 공개 사이트처럼 운영하려면 정식 도메인과 공인 HTTPS 인증서를 사용하는 것이 좋습니다.
